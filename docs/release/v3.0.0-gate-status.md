## GATE I — Detection coverage of new auth events

**Status:** ✅ MET

**Evidence:**
- SELF-003 Sigma rule targets `AuthFailure` events emitted by `services/api/src/auth_service_token.py` and `services/api/src/auth_analyst.py` on every failed authentication attempt.
- Auth failure events are forwarded via the forwarder service to `SIEMHunterSecurity_CL` in Sentinel, with `provenance_tag = "api:auth"` for routing and attribution.
- The detection service compiles SELF-003 to ClickHouse SQL via the `clickhouse-asim-ocsf.yaml` pipeline; the compiled rule fires on `EventID = 'SELF-003'` rows in `detection_hits`.
- Login, logout, and session-expiry events are labeled with distinct `EventID` values (`AUTH_LOGIN`, `AUTH_LOGOUT`) allowing Sentinel hunting queries to scope auth telemetry.
- Audit writes are best-effort / non-blocking (API does not 503 if Sentinel is unreachable), per the design decision recorded in the §7 audit.

**Owner:** detection-engineer
